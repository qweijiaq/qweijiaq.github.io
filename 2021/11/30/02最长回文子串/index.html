最长回文子串(Longest Palindromic Substring)

求一个字符串中最长的回文子串

回文串定义为：aba，abba这样的正反都一样的字符串。

子串和子序列的区别：

  • 子串是连续字符（如ab是abcd的子串），一个有$n$个字符的字符串的子串数量级为$O(n^2)$。
  • 子序列可以是非连续字符（如ab、ac都是abcd的子序列），一个有$n$个字符的字符串的子串数量级为$O(2^n)$。
  1. 双指针算法

双指针算法分为相向型、同向型和背向型三种，求回文子串使用相向型双指针。

伪代码：

for 起点              				O(n)
	for 终点	  					    O(n)
		检测中间的子串是不是一个回文串    O(n)

这种算法的时间复杂度为$O(n^3)$，显然不太合适。

寻找下面实现代码的错误：

class Solution:
    """
    @param s: input string
    @return: the longest palindromic substring
    """
    def longestPalindrome(self, s):
        for length in range(len(s), 0 ,-1):
            for i in range(len(s) - length +1):
                l, r = i, i+length-1
                while l<r and s[l]==s[r]:
                    l += 1
                    r -= 1
                if l>=r:
                    return s[i:i+length]
		return ""
  • 没有异常检测
  • 使用大量单字母的变量命名，意义不明
  • 空格使用不规范
  • 缩进层数过多

改进后的实现代码：

class Solution:
    """
    @param s: input string
    @return: the longest palindromic substring
    """
    def longestPalindrome(self, s):
        if s is None:
            return ""
        
        for length in range(len(s), 0 ,-1):
            for i in range(len(s) - length + 1):
                if self.is_palindrome(s, i, i + length - 1):
                    return s[i: i + length]
 
		return ""

	def is_palindrome(self, s, left, right):
        while left < right and s[left] == s[right]:
            left += 1
            right -= 1
        
        return left >= right
  1. Manacher’s Algorithm

该算法可以在$O(n)$的时间内求的最长回文子串，$n$为字符串长度，是该问题的最优算法，但绝对不是面试官想让实现的算法，因为它会认为你是靠背诵得来的。

  1. 基于中心线枚举的算法

这种算法也是基于双指针的，不过是基于背向型双指针。

当要使得子串为奇数时，中心线有$n$种情况，即字符串的每个字符处；当要使得子串为偶数时，中心线有$n-1$种情况，即字符串的每两个相邻字符的间隔处。

这种算法的时间复杂度为$O(n^2)$，符合面试时的要求，代码实现如下：

class Solution:
    """
    @param s: input string
    @return: the longest palindromic substring
    """
    def longestPalindrome(self, s):
        if s is None:
            return ""
        
        start, longest = 0, 0
        for middle in range(len(s)):
            # odd
            left, right = middle, middle
            while left >= 0 and right < len(s) and s[left] == s[right]:
                left -= 1
                right += 1
            if longest < right -left -1:
                longest = right -left -1
                start = left + 1
            
            # even
            left, right = middle, middle + 1
            while left >= 0 and right < len(s) and s[left] == s[right]:
                left -= 1
                right += 1
            if longest < right -left -1:
                longest = right -left -1
                start = left + 1
                
        return s[start: start + longest]

上述代码存在一个问题，就是odd和even代码的部分存在大量重复，这在工程开发中是不允许的，面试时也容易给面试官留下不好的印象，故需要优化，优化后代码如下：

class Solution:
    """
    @param s: input string
    @return: the longest palindromic substring
    """
    def longestPalindrome(self, s):
        if not s:
            return s
        
        answer = (0, 0)
        for mid in range(len(s))：
        	answer = max(answer, self.get_palindrome_from(s, mid, mid))
        	answer = max(answer, self.get_palindrome_from(s, mid, mid + 1))
        
        return s[answer[1]: answer[0] + answer[1]]
    
    def get_palindrome_from(self, s, left, right):
        while left >= 0 and right < len(s) and s[left] == s[right]:
            left -= 1
            right += 1
        return right - left -1, left + 1
  1. 基于动态规划的算法
class Solution:
    """
    @param s: input string
    @return: the longest palindromic substring
    """
    def longestPalindrome(self, s):
        if not s:
            return s
        
        n = len(s)
        is_palindrome = [[False] * n for _ in range(n)] # [[False] * 3] * 3的写法不可使用，因为只进行了浅拷贝，改变一个元素，它所在列的三个元素均改变
        for i in range(n):
            is_palindrome[i][i] = True
        for i in range(1, n):
            is_palindrome[i][i - 1] = True
            
        start, longest = 0, 1
        for length in range(2, n + 1):
            for i in range(n - length + 1):
                j = i + length - 1
                is_palindrome[i][j] = is_palindrome[i + 1][j - 1] and s[i] == s[j]
                if is_palindrome[i][j] and length > longest:
                    longest = length
                    start = i
                    
        return s[start: start + longest]
面试评价标准

从最长回文子串中我们可以得到的面试考点：

  • 面试不一定会要求你用最优复杂度的算法来解决问题

    • 因此单纯只刷LC之类的OJ，容易让你产生一定要用最优解来解决这样的误区

  • 代码真的不是写出来就可以过

    • 代码质量(Coding Quality)很重要

    • 好的代码质量包括：

      • Bug Free

      • 好的代码风格(Coding Style)，包括变量名命名规范有意义，合理的使用空格，善用空行

      • 容易让人读懂的逻辑。要把复杂的事情用简单的方式，别把简单的事情写复杂了

      • 没有冗余代码

      • 有边界检测和异常处理

快速提高代码质量的十二个技巧

  • Coding Style相关:

    • 二元运算符两边加空格，单元运算符不加空格
    • 花括号和for,if之间要加空格(Java)，圆括号和if之间要加空格
    • 用空行分隔开不同的逻辑块
    • 逗号后面加空格

  • Readability相关

    • 函数名和变量名用1-2个单词作为名称
    • 确保一个函数内部不超过3层缩进(indention)
    • 多用子函数来减少入口函数的代码量
    • 多用continue少用if

  • Bug Free相关

    • 不管有没有可能出问题，都要对入口函数的参数进行异常检测
    • 访问一个下标的时候，一定要确保这个下标不会越界
    • 访问一个对象的属性或者方法时，一定要确保这个对象不是空
    • 不用全局变量
